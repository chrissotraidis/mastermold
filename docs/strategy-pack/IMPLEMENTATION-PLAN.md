2. Spec 04 — add a safe, liquid, dynamically rotated Tier B universe.
3. Spec 05 — make quote-derived execution cost the common source of truth.
4. Spec 02 — add CUSUM event sampling, triple barriers, and per-strategy promotion.
5. Spec 03 — add the Bar Portion timing overlay and contrarian shadow module.
6. Spec 08 — add long-history replay, frozen promotion criteria, and demotion.
7. Spec 06 — add measurement-only CEX gaps and shadow-only Drift shorts.
8. Spec 07 — build the ML pipeline only after at least four weeks of `cusum_tb` shadow evidence; code and parity scaffolding can land earlier, but model activation cannot be declared complete without the evidence window and operator-reviewed model card.

## Goal-based implementation loop

For each spec, repeat this loop without skipping gates:

1. **Derive** — convert every checklist item, constraint, exact reason string, threshold, and `done_when` clause into testable requirements.
2. **Audit** — map each requirement to current code and mark it present, partial, missing, contradictory, or externally evidence-gated.
3. **Implement** — make the smallest cohesive spec-complete slice in checklist order; preserve pure/IO separation and safety boundaries.
4. **Prove locally** — add boundary, regression, persistence, and integration tests matching the whole spec rather than only the happy path.
5. **Run gates** — `bun run typecheck`, `bun test tests`, and `npm run privacy:audit`; for runtime-affecting specs also verify `/api/health` and the affected UI/API surface.
6. **Record evidence** — update this ledger with shipped files, test names, command results, observable state, and any code-vs-spec discrepancy.
7. **Advance or hold** — move to the next spec only when its software requirements are proved. Keep time-dependent promotion, four-week ML history, and operator confirmations visibly pending rather than pretending code completion satisfies them.

## Completion definition

The pack is complete only when every item in all eight implementation checklists has authoritative evidence, all verification gates pass, the review surface states what is live/sample/stubbed/credential-gated/missing, and every strategy is either:

- implemented in shadow with its required evidence loop;
- promoted through the frozen per-strategy gates with operator confirmation; or
- still shadow-only because its documented sample/time gate has not matured, with the unmet evidence named explicitly.

No profitability claim is part of completion. Honest measurement, safe degradation, reproducible replay, and automatic demotion are.
